# Core Purpose & Focus: SharedMinds

**Document Purpose**: A grounding document to anchor development decisions and maintain focus on what makes SharedMinds unique and meaningful.

**Last Updated**: February 2025  
**Status**: Living Document - Reference for All Decisions

---

## Why This App Exists

SharedMinds was created to solve a personal problem: **living with ADHD and needing a system that actually works for how my brain works.**

Traditional productivity tools don't understand:
- Task paralysis when something feels too big
- Executive function challenges that make planning exhausting
- The shame cycle of incomplete tasks and broken streaks
- Time blindness and difficulty estimating effort
- The need for permission, not pressure

**The Core Mission**: Help people with ADHD regain focus and motivation to complete tasks and projects, without the shame and pressure that comes from traditional productivity tools.

---

## What Makes SharedMinds Unique

### 1. **ADHD-First Design Philosophy**

Unlike apps that add "ADHD features" as an afterthought, SharedMinds is built from the ground up with neurodivergent brains in mind:

- **No shame cycles**: No overdue indicators, no streaks that create pressure, no completion percentages that highlight failure
- **Permission-based language**: "You can stop after any step" instead of "Complete all tasks"
- **Cognitive scaffolding**: The system does the executive function work users can't always generate themselves
- **Gentle structure**: Optional planning, flexible completion, incomplete plans are valid

### 2. **Intelligent Task Breakdown**

The core differentiator: **AI-powered task decomposition that transforms overwhelming tasks into manageable micro-steps.**

When "Clean my room" feels impossible, SharedMinds breaks it down into:
1. Look around and identify one area
2. Pick up 3 items that don't belong
3. Gather dirty laundry into one pile
4. Put clean laundry away (even if just in a basket)
5. Clear one surface
6. Take trash out
7. Wipe down one surface
8. Stop and appreciate what you've done

**This is the MVP core feature** - everything else supports this.

### 3. **Time-Aware Life Orchestration**

Planner isn't just a calendar - it's a **temporal planning system** that:
- Organizes by time (Today, This Week, This Month) not by life areas
- Bridges long-term vision to daily actions
- Reflects without tracking (planning support, not data collection)
- Avoids backlogs and catch-up pressure

### 4. **Integrated Life Management**

Unlike single-purpose apps, SharedMinds coordinates:
- **Planning** (Planner) - "What will I do and when?"
- **Execution** (Guardrails) - "How do I do it?"
- **Tracking** (Tracker Studio) - "What did I do?" (separate from planning)
- **Organization** (Spaces) - Personal and shared canvases for everything else

Each system has clear boundaries and distinct purposes, working together without overlap.

### 5. **Adaptive Intelligence**

The system learns and adapts:
- Energy-aware task suggestions
- Pattern recognition for task breakdowns
- Context-aware planning (low energy = fewer steps, more permission)
- Individual preference learning

## The MVP Focus

**What matters most for MVP:**

### Core MVP Features (Must Have)

1. **Intelligent Task Breakdown**
   - AI-powered micro-step generation
   - Manual editing of breakdowns
   - Micro-step completion tracking
   - Basic progress visualization

2. **ADHD-Friendly Task Management**
   - Permission-based language
   - No shame indicators (no overdue, no streaks, no failure metrics)
   - Flexible completion ("done enough" is valid)
   - Energy-aware suggestions

3. **Temporal Planning (Basic)**
   - Today view with intentions
   - Calendar integration
   - Forward-looking only (no tracking in planner)

4. **Personal Spaces (Basic)**
   - Simple widget canvas
   - Core widgets: tasks, calendar, notes

### Nice-to-Have (Post-MVP)

- Advanced Guardrails project management
- Full Tracker Studio with analytics
- Travel planning system
- Advanced AI features
- Social collaboration features
- Complex widget system
- Multi-user households

**The Rule**: If a feature doesn't directly support task breakdown or ADHD-friendly planning, it's post-MVP.

## Core Principles for All Development

### 1. **Reduce Overwhelm**
- Progressive disclosure (show breakdown only when needed)
- No long lists of incomplete items
- Focus on "what's next" not "what's overdue"
- Simple, clear language

### 2. **Support Executive Function**
- Externalize planning (system does the breakdown thinking)
- Reduce initiation barrier (make starting as easy as possible)
- Provide structure when users can't generate it themselves
- Time awareness without pressure

### 3. **Reduce Shame & Pressure**
- No overdue indicators (tasks just resurface gently)
- No streaks or completion percentages
- Permission language always present
- "Done enough" is always valid

### 4. **Adapt to Individual Needs**
- Energy-aware suggestions
- Pattern learning
- Flexible completion
- Optional features (nothing required)

### 5. **Forward-Looking, Not Backward-Tracking**
- Planner asks "What will I do?" not "What did I do?"
- Reflection supports future planning, not data collection
- No catch-up pressure or backlog management

## What Success Looks Like

SharedMinds succeeds when:

✅ Users with ADHD can start tasks that previously felt impossible  
✅ Task paralysis is reduced through intelligent breakdown  
✅ Users feel supported, not judged  
✅ The system adapts to individual energy and cognitive states  
✅ Planning feels helpful, not overwhelming  
✅ Users return because it actually works for their brain  

**The Ultimate Test**: Does this feature help someone with ADHD complete a task they would have otherwise avoided? If yes, it's in scope. If no, it's post-MVP.

## What This Means for Development

### For MVP Development

1. **Prioritize Task Breakdown**: This is the core differentiator. Make it excellent.
2. **Keep It Simple**: Don't build features that aren't essential to the core mission.
3. **Test with ADHD Users**: Every feature should be tested with the target audience.
4. **Iterate on Core**: Better breakdown intelligence > new feature categories.

### For Feature Decisions

**Ask these questions before building:**

1. Does this directly help someone with ADHD complete tasks?
2. Does this reduce overwhelm or increase it?
3. Does this create shame/pressure or reduce it?
4. Is this essential for MVP or can it wait?
5. Does this support the core mission or distract from it?

**If the answer to #1 is "no" or #2/#3 are negative, it's not MVP.**

## The One Sentence to Remember

> **"When a task feels too big, we make it smaller. When you don't know where to start, we show you the first step. When you're stuck, we help you get unstuck."**

This sentence should be true for every feature in the MVP.

## Grounding Questions

When feeling lost or overwhelmed by the scope:

1. **Why am I building this?** To help myself and others with ADHD complete tasks and projects.

2. **What's the core problem?** Task paralysis and executive function challenges that make planning and starting tasks feel impossible.

3. **What's the core solution?** Intelligent task breakdown that provides cognitive scaffolding and permission to start small.

4. **What can wait?** Everything that doesn't directly support task breakdown and ADHD-friendly planning.

5. **What makes this different?** ADHD-first design, intelligent breakdown, permission-based language, adaptive intelligence.

---

**Remember**: You're not building a productivity app. You're building a support system for neurodivergent brains. Every feature should serve that mission.
